## 信号处理与血氧算法流程 (SPO2_Algo.c)

整个血氧仪的信号从采集到最终显示，经过了多个关键步骤。在 F303 端，接收到光电信号后进行滤波、特征提取与参数计算，具体流程如下：

### 1. 数据采集与传输
- **采集 (F310 端)**：主控制器以 100Hz 采样率控制红光(Red)与红外光(IR)交替点亮，通过 ADC 获取光强度的原始信号。
- **传输**：F310 通过 UART (921600波特率) 将打包的原始数据发送给 F303。

### 2. 信号预处理 (IIR 滤波)
- 原始光电信号不可避免地夹杂着高频噪声和 50/60Hz 的工频干扰。
- 在数据推入算法前 (`SPO2_Algo_PushData`)，对 Red 和 IR 信号分别通过 **一阶 IIR 低通滤波器** 进行平滑处理。
- **公式**：`Output = 0.25 * Input + 0.75 * Prev_Output`
- 经过滤波后的数据变得平滑，随后存入容量为 800 个样本点（即 8 秒历史数据）的环形缓冲区 (Ring Buffer) 中。

### 3. 心跳特征提取 (寻找周期)
- 算法每累积满 1 秒数据（100个新点），触发一次滑动窗口计算 (`SPO2_Algo_Process`)。
- 取最近的 4 秒数据窗口，在其中寻找 IR 信号的 **波峰**（局部极大值）。
- 计算相邻波峰之间的时间间隔（点数差），得出平均心跳周期 (`Period`)，并换算出心率 (`Heart Rate = 60 * 100 / Period`)。

### 4. 交直流分量分离 (AC/DC Extraction)
- 在找到的心跳周期长度内，往前取一个完整的周期窗口。
- 在该单一心跳周期内，分别找出 Red 和 IR 信号的 **最大值 (Max)** 和 **最小值 (Min)**。
- **直流量 (DC)**：反映骨骼、肌肉及静脉血等恒定组织的吸收量，计算为 `(Max + Min) / 2`。
- **交流量 (AC)**：反映心脏搏动引起动脉血液容积变化导致的吸收波动，计算为 `Max - Min`。

### 5. 特征比值(R)与血氧(SpO2)计算
- 经典血氧计算基于朗伯-比尔定律，计算特征比值 R。
- **理论公式**：`R = (AC_red / DC_red) / (AC_ir / DC_ir)`
- **代码实现**：为了避免浮点除法并保留精度，将其展开并放大 1000 倍进行整数运算：
  `R_scaled = (AC_red * DC_ir * 1000) / (AC_ir * DC_red)`
- **血氧转换**：使用通用经验公式将 R 值转换为血氧饱和度百分比：
  `SpO2 = 110 - 25 * R` （对应代码中 `110 - (25 * R_scaled) / 1000`）

### 6. 灌注指数(PI)计算与历史平滑
- **灌注指数 (PI)**：反映外周组织的血液灌注情况，公式为 `PI = (AC / DC) * 100%`。
- **滑动平均平滑**：单次计算得出的 SpO2、HR 和 PI 可能存在波动。算法将最近 5 次的合法计算结果存入历史数组，求取滑动平均值，最终输出稳定、可靠的生命体征数据。

## 报警与音频方案

- 报警策略：当 SpO2 < 85 时启动报警，SpO2 ≥ 85 时停止报警。
- 报警测试：UI 菜单新增“Alarm Test”子菜单，可独立开关报警、调整报警音量。
- 音频链路：F303 通过 PA4 输出 DAC 模拟音频，驱动 FM8002A 功放；音量由 AD5160 数字电位器调节。
- 数字音量控制（AD5160）SPI 连接：
  - 5160_CLK → PB13（SCK）
  - 5160_CS  → PB14（NSS）
  - 5160_SDI → PB15（MOSI）
